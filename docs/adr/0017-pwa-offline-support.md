# ADR-0017: PWA with Offline Support via vite-plugin-pwa

## Status
Accepted

## Date
2026-03-18

## Context
NetStack is a client-side-only application (ADR-0002) with no backend. Users in air-gapped networks or field environments need the app to work without internet access. The web manifest already existed with display: standalone, but no service worker was configured — the app required a network connection on every load.

Two distribution approaches were identified:
1. **PWA with service worker** — installable from browser, automatic offline caching
2. **Portable zip archive** — downloadable from GitHub Releases, works via file:// protocol

## Decision
Implement both approaches:

### PWA (vite-plugin-pwa)
- Use `vite-plugin-pwa` with Workbox `generateSW` strategy (zero custom service worker code)
- `registerType: 'prompt'` — users get a toast notification when updates are available, not auto-reload
- Precache all application assets (JS, CSS, HTML, images, icons) via Workbox glob patterns
- Runtime cache Google Fonts with CacheFirst strategy (365-day TTL)
- ReloadPrompt React component with i18n strings in all 4 locales (EN, FR, DE, IT)
- Web manifest generated by plugin (replaces static site.webmanifest)
- Maskable icon (512x512) for Android home screen

### Offline zip (release workflow)
- GitHub Actions workflow triggers on tag push (v*)
- Builds with `NETSTACK_BASE='./'` for relative asset paths (manifest start_url and scope become './')
- Creates zip and tar.gz archives of dist/ directory
- Attaches archives to GitHub Release via softprops/action-gh-release

### Configuration
- `NETSTACK_BASE` environment variable overrides vite `base` path (defaults to '/netstack/')
- Both PWA manifest start_url/scope and HTML asset paths use the same base

## Consequences
- First visit caches ~4.5 MB of assets; subsequent loads are instant from cache
- Users can install NetStack as a standalone app on desktop and mobile
- Update prompt appears as a non-intrusive toast (bottom-right) — users choose when to reload
- Offline zip works from any folder opened in a browser (no web server needed)
- The old static public/site.webmanifest was removed — vite-plugin-pwa generates it
- Service worker adds workbox-window (5.7 KB gzip) to the client bundle
