# v1.1 Requirements — Archive

All 11 v1.1 requirements were delivered in Phases 5–7.

## Rack Configuration

- [x] **RACK-01**: User can define number of racks explicitly (not just derived from totalServers / serversPerRack)
  - *Delivered Phase 6: rack count field in InputForm, creates per-rack editor rows*
- [x] **RACK-02**: User can set different server counts per rack (variable density)
  - *Delivered Phase 6: each rack row has its own server count input (0–500)*
- [x] **RACK-03**: Engine calculates BOM from per-rack configuration array instead of uniform scalars
  - *Delivered Phase 5: SizingInput.racks array replaces totalServers/serversPerRack*

## Server Ports

- [x] **PORT-01**: User can configure frontend (data) port count per server (default 1)
  - *Delivered Phase 6: "Frontend Ports per Server" input (1–2), leaf-facing help text*
- [x] **PORT-02**: User can configure backend (storage/backup) port count per server (default 0)
  - *Delivered Phase 6: "Backend Ports per Server" input (0–2), storage/backup help text*
  - *Note: OOB management ports are always 1 per server, hardcoded — not configurable*
- [x] **PORT-03**: Cable and transceiver counts adjust based on per-server port configuration
  - *Delivered Phase 5: leaf-server cables × frontendPorts, leafPair cables × backendPorts*

## Rack Elevation

- [x] **ELEV-01**: Servers are visible in the rack elevation view with correct U-height
  - *Delivered Phase 7: amber ServerDevice slots with proportional heights (44px/U)*
- [x] **ELEV-02**: User can configure server U-height (1U, 2U, 4U, 8U)
  - *Delivered Phase 7: serverUHeight Select in Physical section of InputForm*
- [x] **ELEV-03**: RACK_CAPACITY_EXCEEDED constraint violation fires when total device U-height exceeds rack size
  - *Delivered Phase 7: per-rack violation in BOM panel with rackNumber/usedU/totalU*

## Uplink Configuration

- [x] **UPLN-01**: User can select number of active uplinks per leaf switch (1 to model maximum)
  - *Delivered Phase 6: "Active Uplinks per Leaf" selector, max computed from SWITCH_CATALOG*
- [x] **UPLN-02**: Oversubscription ratio and cable counts recalculate based on active uplink count
  - *Delivered Phase 5: effectiveUplinks clamp, oversubscription uses effectiveUplinks × uplinkSpeed*
