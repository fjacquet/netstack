    STEP 2 — Implement src/features/topology/fc/utils/buildFCTopologyGraph.ts:

    Signature: `export function buildFCTopologyGraph(bom: FCNetworkBOM): FCTopologyGraphResult`

    Layout strategy: horizontal row of FC switches, evenly spaced at x = i * 200. All switches at y = 0.
    - Fabric A prefix: 'fc-a-' (e.g., 'fc-a-0', 'fc-a-1')
    - Fabric B prefix: 'fc-b-' (e.g., 'fc-b-0', 'fc-b-1')
    - Node type: 'fcSwitchNode' (registered at module level in FCTopologyCanvas in Plan 13-02)

    ISL edges: connect adjacent switches in sequence — switch 0→1, 1→2, ... up to islPortsPerFabric edges total.
    - For N switches and M ISL links: create min(M, N-1) ISL edges (N-1 max for a linear chain; ring when M >= N)
    - Edge ID prefix: 'isl-a-' for Fabric A, 'isl-b-' for Fabric B
    - ISL edge style: dashed, strokeWidth 2, color hsl(213, 94%, 55%) for Fabric A / hsl(32, 95%, 55%) for Fabric B

    CRITICAL — use bom.islPortsPerFabric for ISL edge count, NOT bom.islCables (which is total across both fabrics).

    For usedPorts: hostPortsPerFabric / fabricASwitches (rounded up). totalPorts: same as usedPorts (proxy — FC_SWITCH_CATALOG import is optional; keep it simple).

    Single-switch edge case: if switchCount === 1, return 1 node, 0 ISL edges. Do NOT crash.
