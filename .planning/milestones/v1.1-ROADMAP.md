# v1.1 Enhancements — Milestone Archive

**Shipped:** 2026-03-18
**Tag:** v1.1.0
**Range:** v1.0.0..v1.1.0

## Milestone Goal

Improve the Ethernet calculator with per-rack configuration, configurable server ports, rack capacity validation, and selectable uplinks.

## Phases

### Phase 5: Engine Enhancements (completed 2026-03-17)

**Goal:** Sizing engine correctly calculates BOM from per-rack server arrays, configurable server port counts, and selectable uplink count per leaf switch.

**Plans:**
- 05-01: Evolve SizingInput to racks array (RACK-03) — schema, engine, test migration, store v3 migration
- 05-02: Add port multipliers and active uplinks (PORT-03, UPLN-02) — cable formulas, oversubscription, new tests

**Key deliverables:**
- SizingInput migrated from `totalServers`/`serversPerRack` scalars to `racks: Array<{serverCount}>` array
- `maxServersPerRack` derivation for worst-case OOB and oversubscription calculations
- `portsPerServerFrontend`, `portsPerServerBackend` Zod fields with `.default()` for backward compat
- `activeUplinksPerLeaf` with runtime clamp `effectiveUplinks = min(input, model.uplinkPorts)`
- Cable counts updated: leaf-server × frontendPorts, leafPairs × backendPorts
- Oversubscription denominator uses `effectiveUplinks * uplinkSpeed` (not spine count)
- Persist version 2→3→4, merge() migration for schema evolution
- Dell S5296F-ON added to hardware catalog (96×25G + 8×100G leaf)
- Tests: 159 passing (up from 144 at v1.0)

**Requirements satisfied:** RACK-03, PORT-03, UPLN-02

### Phase 6: Input Configuration UI (completed 2026-03-17)

**Goal:** Engineers can configure explicit rack count with per-rack server density, per-server frontend/backend port counts, and active uplink count directly in the input form.

**Plans:**

- 06-02: RTL test suite for InputForm + human visual verification

**Key deliverables:**

- i18n keys added across EN/FR/DE/IT for all new fields
- FormValues bridge type replaced by direct Zod integration with rack array

## Key Decisions

| Decision | Phase | Rationale |
|----------|-------|-----------|
| racks array replaces scalars | 5 | Foundational RACK-03 change enabling per-rack density |
| maxServersPerRack for OOB/oversub | 5 | Worst-case rack determines constraints |
| Zod .default() for new fields | 5 | Backward-compatible schema evolution without conditional checks |
| Runtime clamp for effective uplinks | 5 | Keeps schema max at 8, enforces model limits at runtime |
| Frontend = data/leaf, Backend = storage/backup | 6 | OOB is always 1 hardcoded (not configurable) |
| SWITCH_U_PER_SERVER_RACK = 3 constant | 7 | Fixed 2-leaf + 1-OOB overhead per server rack |
| Inline style for ServerDevice height | 7 | Tailwind purges dynamic `h-[${n}px]` at build time |
| coveredSlots Set before render loop | 7 | O(n) upfront vs O(n) per slot — avoid redundant find() |
| Violation map key uses rackNumber | 7 | Prevents duplicate React keys for multi-rack violations |
