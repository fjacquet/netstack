## Color

Source: `src/index.css` — CSS custom properties. All values below are light-mode HSL; dark-mode counterparts are defined in `.dark {}` block.

| Role | Light HSL | Dark HSL | Usage |
|------|-----------|----------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` — background | `hsl(240 10% 3.9%)` — background | Page surface, accordion content background |
| Secondary (30%) | `hsl(0 0% 100%)` — card / `hsl(240 4.8% 95.9%)` — muted | `hsl(240 10% 3.9%)` / `hsl(240 3.7% 15.9%)` | Accordion items, FormItem borders, card backgrounds |
| Accent (10%) | `hsl(240 4.8% 95.9%)` — accent | `hsl(240 3.7% 15.9%)` | Checkbox `accent-primary` fill, focus rings on Input fields |
| Warning (amber) | `hsl(38 92% 50%)` border + icon | `hsl(38 95% 64%)` border + icon | AdvisoryAlert border and AlertTriangle icon — ONLY for `variant="warning"` Alert |
| Destructive | `hsl(0 84.2% 60.2%)` | `hsl(0 62.8% 30.6%)` | ViolationAlert `variant="destructive"` — NOT used for advisories |
| Muted foreground | `hsl(240 3.8% 46.1%)` | `hsl(240 5% 64.9%)` | FormLabel, FormDescription, section headings |

**Accent reserved for:** checkbox fill (`accent-primary`), Input focus ring (`ring` token). NOT applied to advisory cards — advisory cards use `warning` amber, not accent.

**Color separation rule (critical):** Advisory cards (`advisories[]`) use `variant="warning"` (amber border). Violation cards (`violations[]`) use `variant="destructive"` (red) or `variant="warning"` for `DAC_DISTANCE_ADVISORY`. This visual distinction is a primary goal of Phase 27.

## Accordion Layout: Geometry Fields Position

```
AccordionItem value="rack-config"   ← "Rack Configuration"
  [existing fields: rackCount, per-rack server counts, rackSize, serverUHeight, switchPositioning]
  ─── geometry fields appended below existing fields ───
  rackPitchMm          [number input]   label: t('sizing.rackPitchMm')
  racksAdjacent        [checkbox]       label: t('sizing.racksAdjacent')
  patchPanelDistanceM  [number input]   label: t('sizing.patchPanelDistanceM')
                       conditional: only renders when racksAdjacent === false
```

The geometry fields are appended at the bottom of the `rack-config` section — below existing rack/server/positioning fields. They do not require a sub-section heading (the `sizing.physicalHeading` key already exists in EN but is not required for accordion section use).

## Interaction Contracts

### Checkbox: racksAdjacent

- **Default state:** checked (true) — `patchPanelDistanceM` field is hidden
- **On uncheck:** `patchPanelDistanceM` field appears immediately below the checkbox (no animation required, conditional render via `form.watch`)
- **On re-check:** `patchPanelDistanceM` field disappears; its value is preserved in form state but ignored by the engine when `racksAdjacent=true`
- **Store sync:** Immediate (boolean — not debounced), follows existing brownfield toggle pattern

### Number Input: rackPitchMm

- **Default value:** 600 (from `SizingInputSchema` default, pre-populated from `input.rackPitchMm`)
- **Constraints:** min=100, max=2000, integer only (type="number")
- **Store sync:** Debounced 150ms — follows existing numeric field pattern (`rackCount`, `portsPerServerFrontend`)
- **FormDescription:** Always visible below the input

### Number Input: patchPanelDistanceM

- **Default value:** 1 (from schema default, pre-populated from `input.patchPanelDistanceM`)
- **Constraints:** min=0, max=100, decimal allowed (not integer — schema uses `z.number()` not `z.number().int()`)
- **Visibility:** Conditional — only rendered when `form.watch('racksAdjacent') === false`
- **Store sync:** Debounced 150ms — follows existing numeric field pattern

### Advisory Section: Render Conditions

- Advisory section heading and cards only render when `bom.advisories.length > 0`
- Advisory cards are non-blocking — they do not prevent the user from proceeding
- Advisory cards are informational only — no dismiss/close action
- Advisory section appears AFTER the violations section (violations take precedence visually)
