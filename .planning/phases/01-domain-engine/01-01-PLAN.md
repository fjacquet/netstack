<task type="auto" tdd="true">
  <name>Task 3: Create Zod schemas for SizingInput, NetworkBOM, ConstraintViolation, and SwitchSpec</name>
  <files>src/domain/schemas/input.ts, src/domain/schemas/bom.ts, src/domain/schemas/catalog.ts, src/domain/schemas/schemas.test.ts</files>
  <behavior>
    - Test: SizingInputSchema rejects totalServers: 0, totalServers: -1, serversPerRack: 0
    - Test: SizingInputSchema rejects non-integer totalServers (e.g., 10.5)
    - Test: SizingInputSchema rejects invalid connectivityType (e.g., '50G')
    - Test: SizingInputSchema accepts valid input {totalServers: 100, serversPerRack: 20, connectivityType: '25G', cableType: 'DAC'}
    - Test: ConstraintViolationSchema accepts OOB_PORT_SATURATION with required/available fields
    - Test: ConstraintViolationSchema accepts SPINE_CAPACITY_EXCEEDED with leafCount/maxLeafs fields
    - Test: ConstraintViolationSchema accepts DAC_DISTANCE_ADVISORY with rackCount/cableType fields
    - Test: NetworkBOMSchema accepts a complete valid BOM object
    - Test: SwitchSpecSchema validates a correct switch spec entry
    - Test: SwitchSpecSchema rejects a switch spec with missing downlinkPorts
  </behavior>
  <action>
1. Create `src/domain/schemas/input.ts`:
   - Define `SizingInputSchema` using Zod v4:
     - totalServers: z.number().int().min(1).max(10_000)
     - serversPerRack: z.number().int().min(1).max(48)
     - connectivityType: z.enum(['25G', '100G'])
     - cableType: z.enum(['DAC', 'AOC', 'fiber'])
   - Export inferred type: `export type SizingInput = z.infer<typeof SizingInputSchema>;`

2. Create `src/domain/schemas/bom.ts`:
   - Import SizingInputSchema from input.ts.
   - Define `ConstraintViolationSchema` as z.discriminatedUnion('code', [...]) with 3 variants:
     - OOB_PORT_SATURATION: { code, required: z.number().int(), available: z.number().int() }
     - SPINE_CAPACITY_EXCEEDED: { code, leafCount: z.number().int(), maxLeafs: z.number().int() }
     - DAC_DISTANCE_ADVISORY: { code, rackCount: z.number().int(), cableType: z.literal('DAC') }
   - Define `NetworkBOMSchema` with all fields: racks, leafSwitches, spineSwitches, oobSwitches, leafSpineCables, serverLeafCables, serverOobCables, oversubscriptionRatio, violations (array of ConstraintViolation), input (SizingInputSchema).
   - Export inferred types for both.

3. Create `src/domain/schemas/catalog.ts`:
   - Define `SwitchSpecSchema` matching the SwitchSpec interface for runtime validation of JSON override entries.
   - Fields: modelId (string), role (enum leaf/spine/oob), downlinkPorts (int, min 0), downlinkSpeedGbE (int, min 0), uplinkPorts (int, min 0), uplinkSpeedGbE (optional int), additionalUplinkPorts (optional int), maxPowerW (number, min 0), typicalPowerW (optional number), switchingCapacityTbps (optional number).

4. Write tests in `src/domain/schemas/schemas.test.ts`:
   - Use `.safeParse()` for all validation tests (not `.parse()` which throws).
   - Cover all behaviors listed above: rejection of invalid inputs, acceptance of valid inputs, discriminated union variants, SwitchSpec validation.
  </action>
