<feature>
  <name>Catalog Override — mergeCatalog</name>
  <files>src/domain/catalog/loader.ts, src/domain/catalog/loader.test.ts</files>
  <behavior>
    ## Valid Override — New Model
    - Input: base catalog + override JSON with a new model 'Z9664F-ON' (valid SwitchSpec)
    - Output: merged catalog has 4 entries (original 3 + new one)
    - The new entry's fields match the override

    ## Valid Override — Replace Existing Model
    - Input: base catalog + override JSON with 'S5248F-ON' having different maxPowerW (700)
    - Output: merged catalog's S5248F-ON has maxPowerW 700, all other fields from override
    - Original base catalog object is NOT mutated (immutability check)

    ## Invalid Override — Missing Required Field
    - Input: override JSON with entry missing downlinkPorts field
    - Output: function throws ZodError (or returns error) with clear message about missing field
    - Base catalog is unmodified

    ## Invalid Override — Wrong Type
    - Input: override JSON with downlinkPorts as string "48" instead of number
    - Output: function throws with validation error
    - Base catalog is unmodified

    ## Empty Override
    - Input: base catalog + empty object {} or undefined
    - Output: returns catalog identical to base (no mutation, new reference)

    ## Malformed JSON String
    - Input: non-parseable JSON string
    - Output: function throws with parse error before Zod validation

    ## Base Catalog Immutability
    - After any mergeCatalog call (valid or invalid), SWITCH_CATALOG retains original values
  </behavior>
