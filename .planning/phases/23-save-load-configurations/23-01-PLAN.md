  <action>
    1. Create `src/domain/schemas/profile.ts`:
       - Define `ProfileSchema` using Zod v4:
         - `id`: z.string().uuid() (generated via crypto.randomUUID())
         - `name`: z.string().min(1).max(100)
         - `mode`: z.enum(['ethernet', 'fc', 'converged'])
         - `topology`: z.enum(['leaf-spine', 'three-tier']).optional() (only meaningful for ethernet/converged)
         - `date`: z.string() (ISO 8601 timestamp)
         - `version`: z.number().int().default(1) (for future migration)
         - `serverCount`: z.number().int().min(0) (summary field for display in list)
         - `inputState`: z.record(z.unknown()) (the full input object from the active store -- typed generically because each mode has a different shape)
       - Export `Profile` type via `z.infer<typeof ProfileSchema>`
       - Export `ProfileListSchema` as `z.array(ProfileSchema)`
       - Do NOT import SizingInput/FCSizingInput/ConvergedSizingInput types here -- keep the schema generic with z.record(z.unknown()) to avoid circular coupling. The service layer handles type safety.

    2. Create `src/domain/profiles/profileService.ts`:
       - `PROFILES_KEY = 'netstack-profiles'` constant
       - `listProfiles(): Profile[]` -- reads localStorage, parses JSON, validates with ProfileListSchema.safeParse, returns sorted by date descending. Returns [] on any error.
       - `saveProfile(params: { name: string; mode: 'ethernet' | 'fc' | 'converged'; topology?: 'leaf-spine' | 'three-tier'; inputState: Record<string, unknown> }): Profile` -- generates id via crypto.randomUUID(), computes serverCount from inputState.racks (sum of serverCount), creates ISO date string, saves to localStorage. If a profile with the same name already exists, replaces it (same id is kept).
       - `loadProfile(id: string): Profile | null` -- finds by id in list, returns null if not found.
       - `deleteProfile(id: string): void` -- filters out by id, writes back to localStorage.
       - All functions use try/catch for localStorage access (private browsing, quota exceeded).

    3. Create `src/domain/profiles/profileService.test.ts`:
       - Use vitest with jsdom environment (add `// @vitest-environment jsdom` pragma)
       - Clear localStorage before each test
       - Test all 7 behaviors listed above
       - Use realistic SizingInput/FCSizingInput fixture data for inputState
  </action>
