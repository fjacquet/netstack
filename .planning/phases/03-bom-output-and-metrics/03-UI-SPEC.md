## Color

### Design Token Mapping for Phase 3 Semantic States

Phase 3 introduces three semantic states not present in Phase 2: oversubscription severity (green/amber/red) and violation alert. These extend the existing token palette.

#### Light Mode

| Role | CSS Variable / Value | Phase 3 Usage |
|------|---------------------|---------------|
| Dominant (60%) | `hsl(var(--background))` = white | BOM panel background, table background |
| Secondary (30%) | `hsl(var(--card))` / `hsl(var(--muted))` | Card surfaces wrapping table sections, metric card backgrounds |
| Accent — primary (10%) | `hsl(var(--primary))` = near-black | Table header text, active selection |
| Oversubscription: safe | `hsl(142 76% 36%)` — green-600 | Ratio badge when <= 3:1 |
| Oversubscription: warning | `hsl(38 92% 50%)` — amber-500 | Ratio badge when > 3:1 and <= 6:1 |
| Oversubscription: critical | `hsl(var(--destructive))` = red-600 | Ratio badge when > 6:1 |
| Violation alert | `hsl(var(--destructive))` | Alert border and icon for OOB saturation and spine capacity exceeded |
| Advisory (DAC distance) | `hsl(38 92% 50%)` — amber-500 | Alert border and icon for DAC distance advisory |
| Muted text | `hsl(var(--muted-foreground))` | "Used / Available" label, port count helper text |
| Border | `hsl(var(--border))` | Table row dividers, card borders |
| Progress bar fill — safe | `hsl(142 76% 36%)` — green-600 | Port utilization bar when < 80% used |
| Progress bar fill — warning | `hsl(38 92% 50%)` — amber-500 | Port utilization bar when >= 80% and < 100% used |
| Progress bar fill — critical | `hsl(var(--destructive))` — red-600 | Port utilization bar when 100%+ used |

#### Dark Mode

Same semantic intent; token resolution is handled by CSS variable fallback in .dark block. The three explicit color values (green-600, amber-500) shift to their dark-mode equivalents:

| State | Dark Value |
|-------|-----------|
| Oversubscription safe | `hsl(142 69% 58%)` — green-400 |
| Oversubscription warning | `hsl(38 95% 64%)` — amber-400 |
| Oversubscription critical | `hsl(var(--destructive))` = dark red per src/index.css |
| Advisory | `hsl(38 95% 64%)` — amber-400 |

Accent reserved for (Phase 3 additions, inheriting Phase 2 list):

- Active tab underline/indicator (inherited Phase 2)
- Primary CTA button background (inherited Phase 2)
- Input ring on focus state (inherited Phase 2)
- Progress bar track background: `hsl(var(--muted))` — NOT accent color
- "View BOM" / "Recalculate" secondary actions — ghost variant, NOT accent fill

Accent is NOT used for: oversubscription badges (they use semantic green/amber/red), table headers, progress bar fills, violation alerts.

### BOM Panel Internal Structure

1. **Oversubscription Metric Card** — always visible at top
   - Single Card component, full-width
   - Shows: ratio value (e.g. "2.4:1"), colored badge, threshold legend
   - Height: auto (single row, ~56px)

2. **Switches Table** — Card wrapping a shadcn Table
   - Heading: "Switches" (Label size, 12px semibold)
   - Columns: Model | Role | Quantity | Port Utilization
   - Rows: one per switch model present in the BOM (Leaf model selected + S5232F-ON + S3248T-ON)
   - Port Utilization column: "used / available" text + Progress bar below

3. **Cables Table** — Card wrapping a shadcn Table
   - Heading: "Cables" (Label size, 12px semibold)
   - Columns: Type | Category | Quantity
   - Rows: Leaf-Spine cables | Server-Leaf cables | Server-OOB cables

4. **Violations Section** — renders only when `violations.length > 0`
   - One shadcn Alert per violation — stacked vertically, gap-sm (8px)
   - OOB_PORT_SATURATION → variant destructive
   - SPINE_CAPACITY_EXCEEDED → variant destructive
   - DAC_DISTANCE_ADVISORY → variant warning (amber, custom variant)
   - Violations section collapses to zero height when empty (no empty card shown)

### Responsive Behavior

| Viewport | BOM Panel Layout |
|----------|-----------------|
| 768px–1279px (tablet) | BOM panel stacks below input panel, full width, each table card is full width |
| 1280px+ (desktop) | BOM panel occupies flex-1 beside input panel, tables fill available width |
| <768px | Not optimized (out of scope per REQUIREMENTS.md) |

## Interaction Contract

### Live Update on Input Change

- BOM panel re-renders whenever `useResultStore` emits a new BOM (module-level subscription — Phase 2 architecture)
- No loading state needed — engine runs synchronously (<1ms)
- Previous BOM values stay visible during the brief re-render cycle (no flash to zero)
- Oversubscription badge color transitions: CSS transition `color 150ms ease, background-color 150ms ease`

### Cable Type Selection

- Cable Type Select in input form (already present from Phase 2) drives the cable SKU category in the Cables table
- The Cables table heading shows the active cable type: "Cables (DAC)", "Cables (AOC)", or "Cables (Fiber)"
- No separate cable type select in the BOM panel — single source of truth is the input form

### Port Utilization Progress Bar

- Width reflects `used / available` ratio as a percentage
- Color changes at thresholds: green below 80%, amber at 80–99%, red at 100%+
- Tooltip on hover shows exact values: "32 used / 48 available (67%)"
- Tooltip uses shadcn Tooltip component, trigger: hover + focus (keyboard accessible)

### Oversubscription Metric Card

- Badge text: "N.N:1" — always one decimal place (e.g. "3.0:1", "6.5:1")
- Badge background: green for <= 3:1, amber for > 3:1 and <= 6:1, red for > 6:1
- Threshold legend: three small labeled chips below the ratio — "Optimal ≤3:1", "Acceptable ≤6:1", "Critical >6:1"
- Active threshold chip is filled; inactive chips use muted outline style

### Violation Alerts

- OOB_PORT_SATURATION: shows required vs available port counts from violation object
- SPINE_CAPACITY_EXCEEDED: shows current leaf count vs maximum supported from violation object
- DAC_DISTANCE_ADVISORY: informational (amber) — not a hard block, shown with warning icon
- Alerts do NOT have a dismiss/close button — they persist while the violation condition exists

### Empty State (no BOM computed)

- Shown when `bom === null` (store initial state before first valid input)
- Renders in the BOM panel area: centered icon + heading + body copy
- Icon: `BarChart3` (lucide, 48px, muted color)
- No loading spinner

## Accessibility Contract

Inheriting all Phase 2 rules plus Phase 3 additions:

- Progress bars use `<progress>` element or role="progressbar" with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Port utilization for {model}: {pct}%"`
- Violation alerts render with `role="alert"` so screen readers announce new violations when inputs change
- Oversubscription badge color uses icon + text label — color is not the sole indicator of state
- Oversubscription badge includes `aria-label`: "Oversubscription ratio: {value}. Status: {optimal/acceptable/critical}"
- Tables use `<caption>` element with section heading text for screen reader context
- Table column headers use `<th scope="col">` — required for data tables
- Tab order within BOM panel: heading → oversubscription card → switches table → cables table → port utilization → violations
- Focus ring: 2px solid `hsl(var(--ring))`, 2px offset — inherited Phase 2 rule
