    ## Test Cases (RED phase -- write these first)

    ### TENG-02: Access switches = 2 per rack
    - 3 racks -> accessSwitches = 6
    - 1 rack -> accessSwitches = 2
    - 10 racks -> accessSwitches = 20

    ### TENG-03: Aggregation switches = ceil(totalAccessUplinks / effectiveAggrDownlinks), min 2
    - 3 racks (6 access), 4 uplinks/access, Z9264F-ON aggr (64 ports, 4 uplinks to core -> 60 effective downlinks): ceil(24/60) = 1 -> clamped to 2
    - 20 racks (40 access), 4 uplinks/access, Z9264F-ON aggr (60 eff down): ceil(160/60) = 3
    - 3 racks, 4 uplinks/access, S5232F-ON aggr (32 ports, 4 uplinks -> 28 eff down): ceil(24/28) = 1 -> clamped to 2

    ### TENG-04: Core switches = ceil(totalAggrUplinks / effectiveCoreDownlinks), min 2
    - 2 aggr switches, 4 uplinks/aggr, Z9332F-ON core (32 downlinks): ceil(8/32) = 1 -> clamped to 2
    - 8 aggr switches, 8 uplinks/aggr, Z9332F-ON core (32 downlinks): ceil(64/32) = 2

    ### TENG-05: Oversubscription per boundary
    - Access-to-aggr: (maxServersPerRack * accessDownlinkSpeed) / (effectiveUplinksPerAccess * accessUplinkSpeed)
    - Aggr-to-core: (effectiveAggrDownlinks * aggrPortSpeed) / (effectiveUplinksPerAggr * aggrPortSpeed)
    - S5248F-ON access (25G down, 100G up), 16 servers/rack, 4 uplinks: (16*25)/(4*100) = 1.0
    - Z9264F-ON access (100G down, 100G up), 16 servers/rack, 4 uplinks: (16*100)/(4*100) = 4.0

    ### TENG-06: Cable BOM
    - serverAccessCables = totalServers * portsPerServerFrontend
    - accessAggrCables = accessSwitches * effectiveUplinksPerAccess
    - aggrCoreCables = aggregationSwitches * effectiveUplinksPerAggregation
    - serverOobCables = totalServers * portsPerServerBackend + accessSwitches
    - vltCables = racks * 2

    ### Additional behaviors
    - OOB switches computed same as Clos: racks * ceil((maxServersPerRack + 2) / 48)
    - Transceivers: sfp28Count for 25G fiber links, qsfp28Count for 100G fiber links, qsfp56ddCount for 400G fiber links, all 0 for DAC/AOC
    - AGGREGATION_CAPACITY_EXCEEDED violation when totalAccessUplinks > aggrSwitches * effectiveAggrDownlinks
    - CORE_CAPACITY_EXCEEDED violation when totalAggrUplinks > coreSwitches * effectiveCoreDownlinks
    - OOB_PORT_SATURATION violation when maxServersPerRack + 2 > 48
    - DAC_DISTANCE_ADVISORY when cableType='DAC' and racks > 8
    - RACK_CAPACITY_EXCEEDED when rack U-height exceeds physical capacity
    - Network racks for aggregation + core + border leaf switches
    - borderLeafSwitches from input (0 if 'none')
    - Symmetric port handling: effectiveDownlinks = totalPorts - activeUplinks (Z-series)
    - S5248F-ON as access: effectiveUplinks = min(activeUplinksPerAccess, uplinkPorts=4)
    - recommendedCableLengthM follows same positioning map as Clos (ToR=2, MoR=1, BoR=2)

Create `src/domain/engine/three-tier-sizing.ts` with:

1. Import SWITCH_CATALOG, ThreeTierSizingInput type, ThreeTierBOM type, ThreeTierConstraintViolation type.

2. Define `export function calculateThreeTierBOM(input: ThreeTierSizingInput): ThreeTierBOM` following the exact same structure as calculateBOM:
   - Catalog lookups: ACCESS = SWITCH_CATALOG[input.accessModel], AGGR = SWITCH_CATALOG[input.aggregationModel], CORE = SWITCH_CATALOG[input.coreModel], OOB = SWITCH_CATALOG['S3248T-ON']
   - Rack count from input.racks.length
   - totalServers from racks.reduce sum
   - maxServersPerRack from Math.max(...racks.map)

3. Implement symmetric port logic:
   - For access switches with dedicated uplinks (S5248F-ON etc): effectiveUplinksPerAccess = min(input.activeUplinksPerAccess, ACCESS.uplinkPorts)
   - For symmetric access switches (Z9264F-ON with uplinkPorts=0): effectiveUplinksPerAccess = min(input.activeUplinksPerAccess, ACCESS.downlinkPorts)
   - Combined: effectiveUplinksPerAccess = min(input.activeUplinksPerAccess, ACCESS.uplinkPorts > 0 ? ACCESS.uplinkPorts : ACCESS.downlinkPorts)
   - Similarly for aggregation: effectiveUplinksPerAggr = min(input.activeUplinksPerAggregation, AGGR.downlinkPorts) since all aggr models have uplinkPorts=0
   - effectiveAggrDownlinks = AGGR.downlinkPorts - effectiveUplinksPerAggr

4. TENG-02: accessSwitches = racks * 2
5. TENG-03: aggregationSwitches = max(2, ceil(accessSwitches * effectiveUplinksPerAccess / effectiveAggrDownlinks))
6. TENG-04: coreSwitches = max(2, ceil(aggregationSwitches * effectiveUplinksPerAggr / CORE.downlinkPorts))
   Note: Core has no upstream, so all ports are downlinks.

7. OOB: oobPortsRequired = maxServersPerRack + 2, oobSwitchesPerRack = ceil(oobPortsRequired / 48), oobSwitches = racks * oobSwitchesPerRack

8. TENG-06 cables:
   - serverAccessCables = totalServers * input.portsPerServerFrontend
   - accessAggrCables = accessSwitches * effectiveUplinksPerAccess
   - aggrCoreCables = aggregationSwitches * effectiveUplinksPerAggr
   - serverOobCables = totalServers * input.portsPerServerBackend + accessSwitches
   - vltCables = racks * 2

9. TENG-05 oversubscription:
   - accessUplinkSpeed = ACCESS.uplinkSpeedGbE ?? ACCESS.downlinkSpeedGbE (for symmetric switches)
   - accessToAggrOversubscription = (maxServersPerRack * ACCESS.downlinkSpeedGbE) / (effectiveUplinksPerAccess * accessUplinkSpeed) -- or 0 if denominator is 0
   - aggrToCoreOversubscription = (effectiveAggrDownlinks * AGGR.downlinkSpeedGbE) / (effectiveUplinksPerAggr * AGGR.downlinkSpeedGbE) -- or 0 if denominator is 0

10. Transceivers:
    - Determine access link speed (ACCESS.downlinkSpeedGbE) and uplink speed
    - sfp28Count: if fiber, 2 * serverAccessCables (only when 25G links)
    - qsfp28Count: if fiber, 2 * accessAggrCables (only when 100G links)
    - qsfp56ddCount: if fiber, 2 * aggrCoreCables (only when 400G links)
    - All 0 for DAC/AOC

11. Network racks: ceil((aggregationSwitches + coreSwitches + borderLeafSwitches) / rackSizeU) or 0

12. Violations array (same pattern as Clos engine):
    - AGGREGATION_CAPACITY_EXCEEDED if needed
    - CORE_CAPACITY_EXCEEDED if needed
    - OOB_PORT_SATURATION
    - DAC_DISTANCE_ADVISORY
    - RACK_CAPACITY_EXCEEDED (per-rack loop, uHeight = parseInt(input.serverUHeight), overhead = 3U for OOB+2 access switches)

13. Cable length map: { ToR: 2, MoR: 1, BoR: 2 }

14. Return the complete ThreeTierBOM object.
