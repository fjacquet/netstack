## Interaction Contract

### Topology Diagram

- Auto-renders when the Topology tab becomes active using the current BOM from `useResultStore`
- Re-renders when BOM changes (node count or positions shift) — `useEffect` on BOM
- Deterministic layout algorithm: fixed column positions per tier, evenly spaced horizontally. Given N spine switches, spine x-positions are `[ (canvasWidth / (N+1)) * i ]` for i=1..N. Same formula for leaf and rack tiers.
- "Fit View" button: calls `reactFlowInstance.fitView({ padding: 0.1, duration: 300 })`
- "Reset Layout" button: re-runs the deterministic layout algorithm (discards user pan/drag)
- Pan: enabled (drag canvas background)
- Zoom: enabled (scroll wheel, pinch, +/- buttons provided by ReactFlow controls)
- Node click: opens a Popover anchored to the node showing port details (model, role, used/available ports)
- Node drag: user can reposition nodes. Custom positions are NOT persisted (reset on BOM change or Reset Layout)
- Edge hover: edge increases from 1.5px to 2.5px width, tooltip shows cable type
- Topology respects `ThemeProvider` dark mode class — canvas background uses `hsl(var(--background))`

### Port Click Popover (Node Detail)

- Trigger: click on any topology node
- Component: shadcn Popover (already installed after Phase 4 add)
- Content:
  - Node model name (monospace heading)
  - Role label
  - Port utilization: "32 / 48 used (67%)" with inline Progress bar
  - Saturation state badge (green/amber/red)
- Popover width: 240px fixed
- Popover dismisses on click outside or Escape key

### Rack Elevation

- Renders the rack selected in the rack selector dropdown
- Rack selector default: "Rack 1" (first rack)
- Rack selector option label format: "Rack {n} — {leafModelA}, {leafModelB}, {oobModel}"
- Auto-updates when BOM changes (new rack count resets selector to Rack 1)
- Device drag-to-reorder: devices are draggable within U-slot rows
  - Drag start: device lifts (box-shadow increases, opacity 0.8)
  - Drag over valid slot: slot border becomes accent color
  - Drop: device snaps to U-slot
  - Invalid drop (onto another device): snaps back to origin with 200ms ease animation
- U-slot click: does nothing (no detail popover on slots in v1)

### Export Actions

- "Export CSV": triggers download of `netstack-bom.csv` — no modal, immediate download
  - If BOM is null: button is disabled with tooltip "Calculate a BOM first"
- "Export PDF": triggers async PDF generation, shows spinner in button for duration, then triggers download of `netstack-report.pdf`
  - PDF library is lazy-loaded — first trigger may show 300–800ms spinner while chunk loads
  - If BOM is null: button is disabled with tooltip "Calculate a BOM first"
- "Print": calls `window.print()`; print stylesheet hides tab bar and shows only active tab content

## Accessibility Contract

Inheriting all Phase 2 and Phase 3 rules plus Phase 4 additions:

- ReactFlow canvas has `aria-label="Network topology diagram"` on the outer container
- Topology nodes have `role="button"` with `aria-label="{model} {role} — click for port details"`
- Node click Popover has `aria-label="Port details for {model}"` on the popover content
- Rack elevation drag-to-reorder provides keyboard alternative: up/down arrow keys move selected device one U-slot, Enter confirms, Escape cancels
- Rack device has `role="button"` with `aria-label="{model} at U{n} — press to select and use arrow keys to move"`
- Rack selector `<Select>` uses shadcn Select component with `aria-label="Select rack to view"`
- Export buttons have `aria-disabled="true"` when BOM is null, not HTML `disabled` — allows tooltip to activate on keyboard focus
- Export PDF button renders `aria-busy="true"` and `aria-label="Generating PDF"` during generation
- Focus ring: 2px solid `hsl(var(--ring))`, 2px offset — inherited Phase 2 rule, applies to all Phase 4 interactive elements
