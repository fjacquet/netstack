       ```
       Domain Layer (pure TS) -> Store Layer (Zustand) -> Feature Layer (React) -> UI Layer (shadcn/Tailwind)
       ```
       One-way import rule: each layer only imports from layers to its left.
    3. **Domain Layer** — Describe:

       - Sizing formulas: Racks, Leafs, Spines, OOB, Cables, Oversubscription (copy exact formulas from CLAUDE.md).

    4. **Store Layer** — Describe:
       - `inputStore` — persisted to localStorage, holds SizingInput.
       - `resultStore` — derived, module-level subscription recomputes BOM on every input change.
       - Pattern: Zustand stores + `useShallow` selectors to prevent infinite render loops.
    5. **Feature Layer** — Describe each feature module:
       - `features/sizing/` — InputForm + BOMPanel

       - `features/rack-elevation/` — Custom rack elevation view with HTML5 drag
       - `features/export/` — CSV, PDF (@react-pdf/renderer lazy-loaded), Print
    6. **Data Flow** — End-to-end: User input -> InputForm -> inputStore -> subscription -> calculateBOM -> resultStore -> BOMPanel/TopologyTab/RackElevationTab/ExportTab
    7. **Technology Stack** — Table: React 19, TypeScript strict, Vite 6, Tailwind v4, shadcn/ui, Zustand, Zod v4, @xyflow/react, @react-pdf/renderer, Vitest
    8. **Key Design Decisions** — Reference the decisions from STATE.md (spine scaling, oversubscription denominator, fail-fast catalog, etc.)

    **docs/USER-GUIDE.md — User Guide**

    Written for end-users (network engineers, procurement). No code. Describe the UI.

    Structure:
    1. **Introduction** — What NetStack does: calculate Dell Leaf-Spine BOM from server count.
    2. **Getting Started** — Open the app URL (GitHub Pages). No installation needed. Works in any modern browser.
    3. **Sizing a Deployment** —
       - Enter Total Servers (1-10,000)
       - Enter Servers per Rack (1-48, OOB alert above 46)
       - Select Connectivity Type (25G or 100G)
       - Select Cable Type (DAC, AOC, or Fiber)
       - Select Leaf Switch Model (S5248F-ON, S5224F-ON, or S5212F-ON)
       - Results update automatically as you type
    4. **Reading the Bill of Materials** —
       - Oversubscription ratio: green (optimal, <=3:1), amber (acceptable, <=6:1), red (critical, >6:1)
       - Switches table: model, role, quantity, port utilization bar
       - Cables table: type, category, quantity
       - Alerts: OOB saturation, spine capacity, DAC distance advisory
    5. **Topology Diagram** —
       - 3-tier layout: spines (top), leafs (middle), racks (bottom)
       - Node colors: blue=leaf, purple=spine, gray=OOB
       - Border glow: green (healthy), amber (>80%), red (saturated)
       - Controls: Fit View, Reset Layout, Legend toggle
       - Click any node for port details
    6. **Rack Elevation** —
       - Select a rack from the dropdown
       - View device placement: OOB at U1, Leaf B at U2, Leaf A at U3
       - Drag devices to rearrange (not persisted)
    7. **Exporting Results** —
       - CSV: downloads immediately, opens in Excel/Google Sheets
       - PDF: generates a multi-page report (takes a moment on first use)
       - Print: Ctrl+P prints the current view cleanly
    8. **Settings** —
       - Theme: light/dark mode toggle (sun/moon icon)
       - Language: EN, FR, DE, IT via dropdown
    9. **Hardware Reference** — Table of 5 Dell switch models (from CLAUDE.md)
    10. **Troubleshooting** —
        - "BOM shows no results" — enter at least 1 server
        - "PDF is slow" — first generation loads the PDF library, subsequent are faster
        - "Topology looks wrong" — click Reset Layout
